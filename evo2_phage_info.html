02   API parameters
Request body fields
Parameter Type Description & phage-specific guidance Recommended
sequence string Seed DNA (ACGT only). For phage work: use the first 200–500 bp of a reference genome or a conserved structural gene (e.g. terminase large subunit). Optionally prepend a taxonomy tag. 500–1000 bp seed
num_tokens integer Tokens to generate. 1 token ≈ 1 nucleotide. For a 30 kb phage genome from a 500 bp seed, set this to ~29,500. Max varies by plan. 5000–30000
temperature float Controls sequence diversity. Lower = more faithful to seed; higher = more novel variants. For synthetic phage design, moderate values balance novelty with plausibility. 0.7–1.0
top_k integer Limits sampling to the top K most likely nucleotides at each position. Lower values produce more deterministic sequences; top_k=1 is greedy decoding. 4 (diversity) or 1 (deterministic)
top_p float Nucleus sampling threshold. Use either top_k or top_p, not both simultaneously. 0.9 (if using top_p)
enable_sampled_probs bool Returns per-position log-probabilities alongside the sequence. Useful for scoring plausibility of each generated nucleotide. true (for scoring runs)
03   taxonomy prompt (key feature for phage work)
Bias generation toward a target phage clade

Prepend a taxonomy tag to your seed sequence to steer Evo 2 toward generating sequences consistent with that lineage. Use NCBI taxonomy names. Leave fields blank or omit if unknown.

taxonomy tag format
|k__Viruses;p__Uroviricota;c__Caudoviricetes;o__Caudovirales;g__[Genus];s__[Species]|
examples — C. acnes phage (PAC-type) seed
# Targeting Propionibacteriales phage clade sequence = ( "|k__Viruses;p__Uroviricota;c__Caudoviricetes;" "o__Caudovirales;g__Pahexavirus;s__Propionibacterium_phage_PA6|" "ATGAAAGCAATTTTCGTACTGAAAGGTTTTGTT..." # your seed here )
✓
Tip: For C. acnes MDR phage design, seed with the large terminase subunit from PAC1 or phiSA039-type sequences. The model trained on curated phage sequences from IMG/VR and can extrapolate novel capsid/tail architectures.

import requests, os, json from pathlib import Path API_KEY = os.environ["NVIDIA_API_KEY"] URL = "https://health.api.nvidia.com/v1/biology/arc/evo2-40b/generate" # --- seed: first 500 bp of a reference C. acnes phage terminase --- SEED = ( "|k__Viruses;p__Uroviricota;c__Caudoviricetes;" "o__Caudovirales;g__Pahexavirus;s__Propionibacterium_phage_PA6|" "ATGAAAGCAATTTTCGTACTGAAAGGTTTTGTT" # truncated for readability ) payload = { "sequence": SEED, "num_tokens": 30000, # ~30 kb novel sequence "temperature": 0.85, "top_k": 4, "enable_sampled_probs": True, } r = requests.post( URL, headers={"Authorization": f"Bearer {API_KEY}"}, json=payload, timeout=300, ) if "application/json" in r.headers.get("Content-Type", ""): result = r.json() seq = result["sequence"] print(f"Generated {len(seq)} bp") Path("phage_candidate_001.fasta").write_text( f">evo2_phage_candidate_001\n{seq}\n" ) elif "application/zip" in r.headers.get("Content-Type", ""): Path("output.zip").write_bytes(r.content) print("Large response saved to output.zip")

# Score a known phage sequence to assess model log-likelihood # Useful for: variant effect prediction, comparing candidate genomes import requests, os, numpy as np API_KEY = os.environ["NVIDIA_API_KEY"] URL = "https://health.api.nvidia.com/v1/biology/arc/evo2-40b/generate" def score_sequence(seq: str) -> float: """Return mean log-probability across all positions.""" r = requests.post( URL, headers={"Authorization": f"Bearer {API_KEY}"}, json={ "sequence": seq[:-1], # seed = all but last nt "num_tokens": 1, "temperature": 1.0, "top_k": 1, "enable_sampled_probs": True, }, timeout=120, ) probs = r.json().get("sampled_probs", []) return float(np.mean(np.log(probs))) if probs else None # Compare WT vs SNP at position 847 (e.g. essential lysin residue) wt_score = score_sequence(wt_sequence) snp_score = score_sequence(snp_sequence) print(f"ΔlogP (SNP - WT): {snp_score - wt_score:.4f}")
05   recommended workflow
From seed to synthesis-ready candidate
🧬
Seed prep
Pull 500 bp from reference phage (NCBI/PATRIC). Add taxonomy tag.
›
⚡
Generate
Call API. Temperature 0.85, top_k 4. Generate 3–5 candidates per seed.
›
🔍
QC filter
GC% check, ORF prediction (Prokka/Pharokka), BLASTn vs. known phages.
›
📊
Score
Rank candidates by mean log-prob. Discard low-probability outliers.
›
🧪
Synthesize
Order top candidates via Twist or IDT. Assemble by Gibson or ARCA.
06   post-generation QC checklist
Sequence-level checks
  • ☐
    GC content: phage genomes typically 35–65%; outliers suggest generation artifacts
  • ☐
    ORF density: run Pharokka or PHANOTATE; expect ≥80% of genome in coding regions
  • ☐
    Codon usage bias: compare to host (C. acnes) with cusp from EMBOSS
  • ☐
    No homopolymer runs: flag runs >8 nt (synthesis difficulty)
  • ☐
    Repeat elements: check for direct/inverted repeats >20 bp with Repeat Finder
Functional annotation checks
  • ☐
    Structural proteins present: terminase large subunit, capsid, tail fiber/spike
  • ☐
    Lytic machinery: endolysin + holin (or spanin for Gram-negative host)
  • ☐
    No lysogeny genes: confirm absence of integrase/repressor for obligate lytic design
  • ☐
    BLASTn vs NCBI phage db: confirm novelty <95% identity over >50% coverage
  • ☐
    No AMR genes: screen against CARD/ResFinder before any in vivo use
07   tips & gotchas
For better phage outputs
✓
Use a functional seed, not random sequence. Start from a real phage gene — Evo 2 performs best when continuing biologically plausible context.
✓
Generate multiple candidates per seed. Run 5–10 generations with different random seeds (vary temperature slightly) and select the best by log-prob score.
✓
Use the Microviridae fine-tuned model (evo2_7b_microviridae via local install) if working with ssDNA phage — it may outperform the 40B base model on that specific clade.
Common pitfalls
⚠
Token ≠ codon: 1 token = 1 nucleotide. A 30 kb genome requires num_tokens: 29500 if seeding with 500 bp. Do not confuse with LLM token counts.
⚠
Large response handling: sequences >~200 tokens return application/zip, not JSON. Always check Content-Type before parsing.
⚠
Free tier rate limits: add time.sleep(2) between batch calls. For production pipelines, upgrade to a paid NGC plan.
